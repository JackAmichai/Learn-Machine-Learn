## 2025-02-14 - FileReader Client-Side DoS via Unbounded File Size
**Vulnerability:** The application was vulnerable to a client-side Denial-of-Service (DoS) because it used `FileReader.readAsText()` on user-uploaded files without enforcing any size limits. This meant a malicious user could upload a multi-gigabyte file, causing the browser to attempt to read the entire file into memory simultaneously, leading to browser freezing, excessive memory consumption, and potentially crashing the user's tab or entire browser.
**Learning:** Even though `FileReader` operates entirely on the client side (in `src/components/Controls.jsx`), the lack of input size validation creates a significant performance and availability vulnerability. Security issues aren't just limited to servers or data leakage; client stability is also a key concern.
**Prevention:** Always enforce strict, reasonable file size limits (e.g., 5MB for configuration JSONs) by checking `file.size` before initiating any `FileReader` operations that load the entire contents into memory.
