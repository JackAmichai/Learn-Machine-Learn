## 2024-03-16 - Accessible Copy-to-Clipboard Feedback Pattern
**Learning:** Adding interactive, absolute-positioned buttons over scrollable content (like `<pre>` code blocks) requires nesting the scrollable area inside a separate wrapper (`.code-block-scroll`). Without this, absolute-positioned elements will scroll along with the overflowing content, confusing the user and making the button inaccessible. Additionally, using a temporary text change ("Copy" -> "Copied!") is an effective, accessible way to provide feedback without needing complex toast notifications inside modals.
**Action:** When adding copy buttons or other floating actions to scrollable areas, always structure the markup with a `position: relative` outer container and a separate `overflow: auto` inner wrapper. Use `navigator.clipboard.writeText` and `setTimeout` for temporary visual confirmation.
